# เว็บไซต์ สงวนดม ไลท์ติ้ง (Sangudom Lighting)

เว็บไซต์ร้านจำหน่ายโคมไฟ — แคตตาล็อกสินค้า + ระบบจัดการสินค้าหลังบ้าน
สร้างขึ้นเพื่อทดแทนบริการเว็บสำเร็จรูป (itopplus) ที่มีค่าใช้จ่ายรายปี

## โครงสร้างไฟล์

```
websangudom/
├── index.html        # หน้าแรก + แคตตาล็อกสินค้า (ค้นหา/กรองหมวดหมู่)
├── about.html        # ความเป็นมา / สาขา / แผนที่ / ติดต่อ
├── admin.html        # หน้าเข้าสู่ระบบ + จัดการสินค้า (เพิ่ม/แก้/ลบ)
├── css/
│   └── style.css     # สไตล์ทั้งหมด
├── js/
│   ├── data.js       # ข้อมูลสินค้าตัวอย่าง + หมวดหมู่
│   ├── main.js       # แสดงแคตตาล็อก, ค้นหา, popup รายละเอียด
│   └── admin.js      # ระบบ login + จัดการสินค้า
├── images/           # โฟลเดอร์เก็บรูปสินค้า
├── .gitignore
└── README.md
```

## วิธีเปิดดูเว็บ

ดับเบิลคลิกไฟล์ `index.html` เปิดในเบราว์เซอร์ได้เลย

## หน้าจัดการสินค้า (เดโม)

เข้าที่ `admin.html`
- ผู้ใช้: `admin`
- รหัส: `1234`

> ⚠️ เวอร์ชันนี้เป็น **เดโม** — ข้อมูลเก็บในเครื่อง (localStorage)
> และ login เป็นแบบง่ายๆ ยังไม่ปลอดภัยสำหรับใช้งานจริง

## แผนพัฒนาต่อ (อนาคต)

| ส่วน | ตอนนี้ (เดโม) | เป้าหมายจริง |
|------|---------------|--------------|
| เก็บข้อมูลสินค้า | localStorage (เฉพาะเครื่อง) | **Firebase Firestore** (ซิงค์ทุกเครื่อง) |
| ระบบ Login | user/pass ในโค้ด | **Firebase Authentication** (ปลอดภัย) |
| รูปภาพ | โฟลเดอร์ images/ ใน Git | images/ ก่อน → ย้าย Cloudinary ถ้าจำเป็น |
| โฮสต์เว็บ | เปิดไฟล์ในเครื่อง | **GitHub Pages / Netlify** (ฟรี) |

### ขั้นตอนต่อ Firebase (สรุปคร่าวๆ)
1. สมัคร Firebase (ฟรี ไม่ต้องผูกบัตรสำหรับ Firestore + Auth)
2. สร้างโปรเจค → เปิด Firestore Database + Authentication
3. เอา config มาใส่ในเว็บ แล้วแก้ `admin.js` ให้อ่าน/เขียนข้อมูลผ่าน Firestore
4. ข้อมูลสินค้าจะซิงค์ทุกเครื่องอัตโนมัติ login เครื่องไหนก็แก้ได้
